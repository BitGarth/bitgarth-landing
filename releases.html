Release notes

What's changed.

A field log of every BitGarth release, newest first.

0.1.7 #

Latest

20 June 2026

Highlights

The free tier now has a clear limit on synced accounts, and BitGarth prompts you to upgrade when you reach it.

New

  • Account limits on the free tier, with prompts on inactive accounts that link straight to the Upgrade page (renamed from Payments).
  • Each paid plan’s terms now appear beside its card, with yearly pricing in whole dollars.

Fixed

  • Wallet lookups now surface CoinGecko rate-limit messages instead of a generic internal error.
  • More reliable manual-asset price lookups, including cached results when no CoinGecko API key is set, and no reuse of stale price details.
  • UI polish: warmer notice surfaces, quieter account-list accents, and a stray section number removed from the Wallets and Upgrade pages.

0.1.6 #

17 June 2026

Highlights

Wallet reports now value your holdings using real market prices pulled from data providers, instead of relying only on manual or zero prices.

New

  • Provider-sourced daily prices in wallet reports, with the price source shown.
  • Year-dial control to pick the date range on the Wallets page.
  • Transactions now default to the current year with simpler filters.

Fixed

  • Wallet report prices ignore zero balances so empty assets don’t skew values.
  • Manual asset assertions carry forward correctly across years.
  • Identicon line glyphs render correctly again.

0.1.5 #

14 June 2026

Highlights

Wallet values can now use real market prices from CoinGecko, with your explicit consent and an optional CoinGecko API key for higher limits.

BitGarth now tells you when a newer version is available, and warns when the running build has drifted from the server.

New

  • CoinGecko price fetching for wallet values, controlled from the navbar and Settings with a clear consent toggle.
  • Historical daily price backfill so past-dated wallet values reconcile over time.
  • Discover and add manual assets from the CoinGecko catalog, with a live search count of matching assets.
  • In-app upgrade awareness: update-status banner, channel-aware upgrade notice, and an automatic throttled check after login.
  • Server/client build-drift detection that prompts a reload when they diverge.
  • hledger export now includes network-fee accounts, a root include tree, and signed balance deltas for correct asset postings.
  • Settings “Security” tab renamed to “Account”, plus a navbar identity dropdown with deep links and larger botanical-ledger identicons.

Fixed

  • BTC ledger values repair on database open and exclude network fees so balances and exports stay correct.
  • Manual assets no longer duplicate, and CoinGecko tickers are not persisted as display glyphs.
  • CoinGecko API keys are redacted from logs and the loader fails closed on a bad key.
  • The build endpoint is gated to web server contexts and the drift notice clears once the server build matches.

0.1.4 #

5 June 2026

Highlights

Manual assets are now backed by a CoinGecko-sourced asset catalog, so you can search and add a far wider range of coins and tokens.

Wallet reports gain a Prices section where you can set your own price for an asset and see it applied to your holdings.

New

  • Searchable manual-asset picker (combobox) ordered by market-cap rank, with many new networks including Monero, Tezos, Zcash, Dash and Polkadot.
  • User price overrides, shown as inline price chips in desktop and mobile wallet reports.
  • Etherscan API key visibility: a set/unset badge in Settings and warnings on the Wallets and Transactions pages when a key is missing.
  • Migration of legacy custom accounts into the new manual-asset model, with confirmation options on the Account page.
  • Wallet Data Export V4, which now backs up and restores your stored API keys.
  • Settings sections are deep-linkable via a query parameter.

Fixed

  • Manual ERC-20 assets are no longer rejected unless they were USDC on Ethereum.
  • Imported xpub addresses are bootstrapped correctly and empty HD sync work sets are preserved.
  • Stored API keys are kept in a dedicated table and blank keys are rejected.
  • Corrected the Cardano ledger model and the Uniswap unit code.

Newest release at the top · older versions stay anchored below.
